# 轨迹算法说明

## 1. 目标

算法模块用于支撑轨迹清洗、停留点识别、异常速度检测、漂移点检测和轨迹抽稀。第一阶段应优先实现简单、可解释、可测试的算法版本。

## 2. 输入点模型

```text
TrajectoryPoint
- eventId
- entityId
- longitude
- latitude
- speed
- direction
- accuracy
- eventTime
```

算法模块不应直接依赖数据库实体、Controller 或消息队列对象。

## 3. 轨迹清洗

处理顺序：

1. 丢弃经纬度非法点。
2. 丢弃缺失实体或事件时间的点。
3. 按 `eventId` 去重。
4. 按 `eventTime` 排序。
5. 标记速度异常点。
6. 标记疑似漂移点。

经纬度校验：

```text
-180 <= longitude <= 180
-90 <= latitude <= 90
```

## 4. 距离计算

MVP 阶段使用 Haversine 公式计算两点球面距离。

输出单位：米。

测试场景：

- 相同点距离为 0。
- 上海附近短距离计算误差可接受。
- 跨城市距离大于短距离阈值。

## 5. 异常速度检测

规则：

```text
distance_meter = distance(pointA, pointB)
duration_hour = (pointB.eventTime - pointA.eventTime) / 3600
speed_kmh = distance_meter / 1000 / duration_hour
```

当 `speed_kmh > threshold` 时标记为速度异常。

建议默认阈值：

| 实体类型 | 阈值 |
| --- | --- |
| `PERSON` | 20 km/h |
| `VEHICLE` | 180 km/h |
| `DEVICE` | 120 km/h |

## 6. 漂移点检测

第一阶段采用三点规则：

```text
A -> B -> C
```

当满足以下条件时，B 可标记为疑似漂移点：

1. A 到 B 的距离超过漂移距离阈值。
2. B 到 C 的距离超过漂移距离阈值。
3. A 到 C 的距离明显小于 A 到 B 和 B 到 C。
4. 时间间隔较短。

建议默认参数：

| 参数 | 默认值 |
| --- | --- |
| 漂移距离阈值 | 500 米 |
| 回归距离阈值 | 100 米 |
| 最大时间间隔 | 120 秒 |

## 7. 停留点识别

规则：

1. 按实体和事件时间排序轨迹点。
2. 从起点开始累积窗口。
3. 如果窗口内所有点距离中心点不超过停留半径，并且持续时间超过阈值，则生成停留点。
4. 停留点经纬度取窗口点位平均值。

默认参数：

| 参数 | 默认值 |
| --- | --- |
| 停留半径 | 100 米 |
| 停留时长 | 10 分钟 |
| 最少点数 | 3 |

输出：

```text
StayPoint
- entityId
- longitude
- latitude
- startTime
- endTime
- durationSeconds
- pointCount
- confidence
```

## 8. 轨迹抽稀

第一阶段使用 Douglas-Peucker 算法。

使用场景：

- 历史轨迹回放。
- 地图展示大量点位。
- 降低前端渲染压力。

参数：

| 参数 | 说明 |
| --- | --- |
| `tolerance` | 抽稀容差，单位米 |
| `maxPoints` | 最大返回点数 |

约束：

- 必须保留首点和尾点。
- 必须保持时间顺序。
- 不应修改原始轨迹数据。

## 9. 围栏判断

圆形围栏：

- 判断点到圆心距离是否小于等于半径。

多边形围栏：

- 使用 PostGIS `ST_Contains` 或 `ST_Within`。

事件类型：

- 进入围栏。
- 离开围栏。
- 停留超时。

## 10. 可测试性要求

算法测试必须覆盖：

- 经纬度非法值。
- 重复事件。
- 乱序点排序。
- 零时间间隔。
- 超高速异常。
- 三点漂移。
- 停留点临界半径。
- 轨迹抽稀保留首尾点。

## 11. 风险提示

- 执行风险：不要在第一阶段引入复杂机器学习模型，规则算法更适合演示和验证。
- 错误风险：速度计算必须处理时间差为 0 的场景。
- 准确性风险：停留点识别参数需要根据实体类型调整。
- 前端风险：抽稀只用于展示，不能替代原始轨迹存储。
